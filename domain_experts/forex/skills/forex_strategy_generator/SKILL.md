# Forex Strategy Generator Skill

## 概述

forex_strategy_generator 是一个用于生成 EUR/USD H4 空头策略参数包的 Blockcell Skill。它通过 AI 分析市场并生成完整的策略参数 JSON 文件，供 MT4 EA 使用。

## 功能特性

- ✅ 生成符合 MT4 EA 要求的完整参数包
- ✅ 自动生成唯一版本号（格式：YYYYMMDD-HHMM）
- ✅ 使用 UTC 时间确保时区一致性
- ✅ 输出 ISO 8601 格式的时间字符串
- ✅ 保存到指定路径供 EA 读取
- ⚠️ V1 版本使用默认参数值（未集成 AI 分析）

## 使用方法

### 手动触发

```bash
# 方法 1：通过 CLI 触发（推荐）
blockcell run msg "生成外汇策略参数"

# 方法 2：使用配置脚本（创建一次性 Cron 任务）
cd domain_experts/forex/skills/forex_strategy_generator
./setup_cron.sh test

# 注意：Gateway 不提供直接调用 Skill 的 HTTP API
# 需要通过 CLI 或创建一次性 Cron 任务来手动触发
```

### 定时触发

配置 Cron 任务，每天 UTC 06:00 自动生成参数包。

**快速配置（推荐）：**

```bash
# 使用配置脚本
cd domain_experts/forex/skills/forex_strategy_generator
./setup_cron.sh daily
```

**手动配置：**

```bash
# 方法 1：通过 CLI
blockcell run msg "创建定时任务：每天UTC 06:00生成外汇策略参数"

# 方法 2：通过 Gateway API
# 注意：Cron 表达式使用 6 段格式（秒 分 时 日 月 周）
curl -X POST http://localhost:18790/v1/cron \
  -H "Content-Type: application/json" \
  -d '{
    "name": "forex_param_generator_daily",
    "message": "生成外汇策略参数包",
    "cron_expr": "0 0 6 * * *",
    "skill_name": "forex_strategy_generator",
    "deliver": false
  }'
```

**详细配置说明：**

请参考 [CRON_SETUP.md](CRON_SETUP.md) 获取完整的 Cron 配置指南，包括：
- 多种配置方法
- Cron 表达式说明
- 时区配置
- 监控与告警
- 故障排查

## 部署要求

### 时区配置

Blockcell 必须在 UTC 时区环境中运行，以确保时间计算正确：

**Linux/Mac:**
```bash
# 设置环境变量
export TZ=UTC
blockcell run

# 或修改系统时区
sudo timedatectl set-timezone UTC
```

**Docker:**
```dockerfile
ENV TZ=UTC
```

**验证时区:**
```bash
date +%Z  # 应该输出 UTC
```

## V1 限制

当前 V1 版本的限制：

1. **未集成 AI 分析**：forex_news、forex_analysis、forex_strategy 技能尚未实现，当前使用默认参数值
2. **固定参数值**：entry_zone、invalid_above、tp_levels 等使用预设值
3. **无新闻窗口**：news_blackout 默认为空数组
4. **时间计算**：时间转换已实现闰年判断和实际月份天数计算

## 故障排查

### 文件保存失败

**问题**：无法保存参数包文件

**解决方案**：
1. 检查文件路径是否存在：`domain_experts/forex/ea/`
2. 检查文件权限：确保 Blockcell 有写入权限
3. 检查磁盘空间：确保有足够的磁盘空间

### 时间格式错误

**问题**：生成的时间格式不正确

**解决方案**：
1. 确认 Blockcell 运行在 UTC 时区
2. 检查 `TZ` 环境变量：`echo $TZ`
3. 重启 Blockcell 使时区配置生效

### EA 无法读取参数包

**问题**：MT4 EA 报告参数文件不存在

**解决方案**：
1. 确认文件路径配置正确
2. 检查 EA 的 `ParamFilePath` 输入参数
3. 确认文件已成功生成：`ls -la domain_experts/forex/ea/signal_pack.json`
