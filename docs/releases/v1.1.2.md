# Release Notes - v1.1.2

- 发布日期: 2026-03-13
- 对比范围: `v1.1.1...v1.1.2`
- 发布标签: `v1.1.2`
- 发布提交: `66761d84c20649638f011d7b1ec9b5b48809621f`

## 概览

v1.1.2 以“同步上游、MT4 EA 能力完善、稳定性提升”为核心。

- 变更统计: 107 files changed, 16018 insertions(+), 1037 deletions(-)

## 主要更新

### 1. 同步上游 blockcell 主线

- 已将当前开发分支同步到 `origin/main` 最新基线（当前可见 `origin/main` 指向提交 `15a4d52`）。
- 合入了主线在 provider、runtime、gateway、cron、文档等方向的持续改进。

重点包括:

- OpenAI Responses 支持: `crates/providers/src/openai_responses.rs`
- 路径访问策略能力: `crates/core/src/path_policy.rs`
- 路径访问策略文档: `docs/22_path_access_policy.md`, `docs/en/22_path_access_policy.md`
- WeCom 通道增强: `crates/channels/src/wecom.rs`

### 2. MT4 Forex EA 核心能力落地

- 完成并集成 EA 关键模块:
  - `TimeUtils`, `ParameterLoader`, `StrategyEngine`
  - `OrderExecutor`, `RiskManager`, `PositionManager`
  - `Logger`, `TimeFilter`
- 新增/完善 EA 文档与协议说明:
  - `domain_experts/forex/ea/docs/BACKTEST_REPORT_TEMPLATE.md`
  - `domain_experts/forex/ea/docs/PARAMETER_PROTOCOL.md`

### 3. 稳定性与安全修复

- 修复 `ServerUTCOffset` 编译问题。
- 修复待执行信号路径有效期校验与错误信息透出。
- 增加 `tp_levels` / `tp_ratios` 长度一致性校验。
- 修复多项运行时安全问题并补充经验文档。

### 4. 运维与自动化

- 新增 MT4 runner 日志清理脚本，支持按保留天数和体积阈值清理:
  - `domain_experts/forex/ea/scripts/cleanup_runner_logs.ps1`
- 新增定时任务注册脚本:
  - `domain_experts/forex/ea/scripts/register_runner_log_cleanup_task.ps1`

## 影响范围

- 代码与文档主要变更目录:
  - `domain_experts/forex/**`
  - `crates/**`
  - `bin/blockcell/**`
  - `docs/**`
  - `webui/**`
- `.gitignore` 已补充 MT4 runner 相关产物忽略规则。

## 升级建议

1. 使用 MT4 runner 的环境，建议启用日志清理定时任务，避免日志长期堆积。
2. 使用 OpenAI provider 的场景，建议回归验证 `openai_responses` 相关调用链路。
3. 依赖路径访问控制的部署，建议按 `docs/22_path_access_policy.md` 复核现有配置。
